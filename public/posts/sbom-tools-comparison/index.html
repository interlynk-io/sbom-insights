Introduction#

Generating accurate SBOMs is crucial for software supply chain security. With numerous tools available, choosing the right one can be challenging. This post compares popular SBOM generation tools across various dimensions.

Detailed Tool Analysis#

Syft by Anchore#

Strengths#

  • Excellent container image support
  • Multiple output formats (SPDX, CycloneDX, Syft JSON)
  • Fast scanning speed
  • Great CI/CD integration

Installation#

# macOS
brew install syft

# Linux
curl -sSfL https://raw.githubusercontent.com/anchore/syft/main/install.sh | sh -s -- -b /usr/local/bin

# Docker
docker run --rm -v /var/run/docker.sock:/var/run/docker.sock \
  anchore/syft:latest <image>

Usage Examples#

# Scan a container image
syft nginx:latest -o cyclonedx-json > nginx-sbom.json

# Scan a directory
syft dir:. -o spdx-json > app-sbom.json

# Scan with multiple formats
syft . -o cyclonedx-json=sbom.cdx.json -o spdx-json=sbom.spdx.json

Best For#

  • Container-based applications
  • Multi-language projects
  • Quick SBOM generation

Trivy by Aqua Security#

Strengths#

  • Integrated vulnerability scanning
  • Comprehensive language support
  • License detection
  • Misconfiguration scanning

Installation#

# macOS
brew install trivy

# Linux
curl -sfL https://raw.githubusercontent.com/aquasecurity/trivy/main/contrib/install.sh | sh -s -- -b /usr/local/bin

# Docker
docker run --rm aquasec/trivy <command>

Usage Examples#

# Generate SBOM with vulnerability scan
trivy image --format cyclonedx --output result.json nginx:latest

# Scan filesystem
trivy fs --format spdx-json --output sbom.json .

# Include license information
trivy image --scanners license,vuln --format cyclonedx nginx:latest

Best For#

  • Security-focused teams
  • Combined SBOM and vulnerability scanning
  • Container security

CycloneDX Generator Tools#

Strengths#

  • Native CycloneDX format
  • Language-specific accuracy
  • Extensive ecosystem
  • Plugin architecture

Available Generators#

# Node.js projects
npm install -g @cyclonedx/cyclonedx-npm
cyclonedx-npm --output-file bom.json

# Python projects
pip install cyclonedx-bom
cyclonedx-py -o sbom.json

# Java projects (Maven)
mvn org.cyclonedx:cyclonedx-maven-plugin:makeAggregateBom

# .NET projects
dotnet tool install --global CycloneDX
dotnet CycloneDX <path> -o sbom.json

Best For#

  • Single-language projects
  • Teams standardized on CycloneDX
  • Maximum accuracy for specific ecosystems

cdxgen#

Strengths#

  • Universal tool for multiple languages
  • Automatic language detection
  • Container support
  • Evidence collection

Installation#

# npm
npm install -g @cyclonedx/cdxgen

# Docker
docker run --rm -v $(pwd):/app ghcr.io/cyclonedx/cdxgen -o /app/bom.json

Usage Examples#

# Auto-detect and generate
cdxgen -o bom.json

# Specific language
cdxgen -t python -o bom.json

# With evidence
cdxgen --evidence -o bom.json

# Container scan
cdxgen -t docker nginx:latest -o container-bom.json

Best For#

  • Polyglot environments
  • Automated pipelines
  • Quick prototyping

Integration Examples#

GitHub Actions#

name: Generate SBOM
on: [push]
jobs:
  sbom:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Generate SBOM with Syft
        uses: anchore/sbom-action@v0
        with:
          format: cyclonedx-json
          output-file: sbom.json

      - name: Upload SBOM
        uses: actions/upload-artifact@v3
        with:
          name: sbom
          path: sbom.json

GitLab CI#

generate-sbom:
  image: aquasec/trivy:latest
  script:
    - trivy fs --format cyclonedx --output sbom.json .
  artifacts:
    paths:
      - sbom.json
    reports:
      cyclonedx: sbom.json

Jenkins Pipeline#

pipeline {
    agent any
    stages {
        stage('Generate SBOM') {
            steps {
                sh 'docker run --rm -v $(pwd):/src ghcr.io/cyclonedx/cdxgen -o /src/sbom.json'
                archiveArtifacts artifacts: 'sbom.json'
            }
        }
    }
}

Choosing the Right Tool#

Decision Tree#

  1. Container-focused? → Syft or Trivy
  2. Need vulnerability scanning? → Trivy
  3. Single language project? → Language-specific CycloneDX tool
  4. Polyglot project? → cdxgen or Syft
  5. License compliance focus? → SPDX Tools
  6. Speed critical? → Syft or cdxgen

Recommendations by Use Case#

For Enterprises#

Primary: Trivy + CycloneDX language-specific tools Rationale: Security focus with accuracy

For Open Source Projects#

Primary: SPDX Tools or cdxgen Rationale: License clarity and broad compatibility

For DevOps Teams#

Primary: Syft Rationale: Speed and container support

For Compliance Teams#

Primary: SPDX Tools + Trivy Rationale: Standards compliance and license detection

Advanced Configuration#

Syft Configuration#

# .syft.yaml
output:
  - cyclonedx-json
  - spdx-json
catalogers:
  enable:
    - javascript-package-cataloger
    - python-package-cataloger
  disable:
    - rust-cargo-lock-cataloger

Trivy Configuration#

# trivy.yaml
scan:
  security-checks:
    - vuln
    - license
format: cyclonedx
severity:
  - CRITICAL
  - HIGH

Common Pitfalls and Solutions#

Problem: Incomplete Dependencies#

Solution: Use multiple tools and merge results

# Generate with multiple tools
syft . -o cyclonedx-json=syft-sbom.json
cdxgen -o cdxgen-sbom.json

# Merge using sbom-utility
sbom-utility merge syft-sbom.json cdxgen-sbom.json -o merged-sbom.json

Problem: Missing License Information#

Solution: Combine with license scanning tools

# Use Trivy for license detection
trivy fs --scanners license --format cyclonedx .

Problem: Large SBOM Files#

Solution: Use compression and filtering

# Compress output
syft . -o cyclonedx-json | gzip > sbom.json.gz

# Filter components
syft . --cataloger javascript-package-cataloger

Conclusion#

No single tool is perfect for all scenarios. The best approach often involves:

  1. Primary Tool: Choose based on your main use case
  2. Validation: Use a secondary tool for verification
  3. Automation: Integrate into CI/CD pipeline
  4. Storage: Implement SBOM management solution

Start with Syft or Trivy for general use, then specialize based on your specific needs. Remember that SBOM generation is just the first step - you’ll also need tools for storage, analysis, and monitoring.
